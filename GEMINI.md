### 📌 프로젝트 검수 요청 프롬프트

너는 **NestJS 실무 경험이 풍부한 시니어 백엔드 개발자**다.

아래에 제공하는 프로젝트는 **NestJS 학습 목적**으로 만든 **간단한 게시판 프로젝트**다.

### 🔍 검수 범위

- 프로젝트 **전체 파일 구조와 코드**를 종합적으로 분석해라.
- 단순 문법 설명이 아니라 **실무에서 실제로 사용하는 방식 기준**으로 평가해라.

### ✅ 중점적으로 봐줬으면 하는 부분

1. **NestJS 아키텍처 및 패턴**
   - Controller / Service / Module 분리 적절성
   - DTO, Entity, Repository 사용 방식
   - Dependency Injection 활용 여부
   - 실무에서 자주 쓰는 NestJS 패턴 적용 가능성
2. **타입 안정성**
   - `any` 사용 여부를 집중적으로 검토해라.
   - `any`가 사용된 부분은
     → 왜 문제인지
     → 어떤 타입으로 대체하는 게 실무적으로 적절한지
     → 예시 코드와 함께 제안해라.
   - DTO, Response 타입, Service 반환 타입까지 꼼꼼히 확인해라.
3. **코드 품질 및 유지보수성**
   - 네이밍 컨벤션
   - 중복 코드 제거 가능성
   - 책임 분리(SRP)가 잘 되어 있는지
   - 추후 기능 확장 시 문제가 될 수 있는 구조인지
4. **실무 관점 개선 사항**
   - 현재는 단순 게시판이지만
     실무 서비스라고 가정했을 때 **반드시 개선해야 할 부분**
   - Validation, Exception Filter, Pipe, Interceptor 적용 여부
   - Config 관리, 환경 변수 처리 방식
   - 로그 처리 방식 (Logger 사용 여부)

### 🛠️ 개선 제안 방식

- 문제가 되는 부분을 **구체적인 파일/코드 단위로 지적**해라.
- 단순 지적이 아니라 **왜 개선해야 하는지 이유**를 함께 설명해라.
- 가능하면 **Before / After 코드 예시**를 제시해라.
- “학습용”이 아닌 **실무에서 실제로 많이 쓰는 방식** 위주로 제안해라.

### 🎯 목표

이 프로젝트가 “NestJS를 공부한 흔적이 보이는 프로젝트”가 아니라
**“실무 주니어 백엔드 개발자가 작성한 코드”처럼 보이도록** 개선 방향을 제시하는 것이 목표다.
